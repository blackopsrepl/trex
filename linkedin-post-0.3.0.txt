🦖 trex 0.3.0 released!

The fast tmux session manager now watches your AI coding assistants.

trex now scans your system for running AI coding agents and displays them in a dedicated dashboard at the top of the interface:

┌─ RUNNING AGENTS ─────────────────────────────────────────┐
│ ▶ claude:my-project ●    ⏸ opencode:api-server ●         │
│ ⏸ claude:frontend ● (claude, zoyd)                       │
└──────────────────────────────────────────────────────────┘

You will be able to see, at a glance:
- Which AI agents are running and in which projects
- Real-time activity state: ▶ actively working or ⏸ waiting for input
- Whether they're running inside tmux (●) or standalone (○)
- Hierarchical agent spawning — when Claude spins up sub-agents, you see them grouped under the parent
- Activity refreshes every 100ms, process list every 2s

You can navigate directly to your agents!
Press `k` to move focus to the agent box, then `Enter` to jump straight into that agent's tmux session. No more hunting through tabs to find where Claude is running.

When you expand a session or use preview mode, the dashboard filters to show only agents in that specific session.

Written in Rust. Zero config. Vim-style keybindings.
